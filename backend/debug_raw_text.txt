🟢 PHẦN 1: J2EE & JSP (NỀN TẢNG)

1.1. JSP (JavaServer Pages)

Định nghĩa: Là công nghệ phía Server giúp tạo trang web động. JSP kết hợp mã HTML với các thẻ Java. Khi thực thi, Web Container sẽ biên dịch JSP thành một Servlet.

Các loại thẻ JSP (Directives & Scripting):

<%@ ... %> (Directive): Cấu hình trang (import thư viện, định nghĩa kiểu nội dung).

<% ... %> (Scriptlet): Chứa mã Java thực thi logic.

<%= ... %> (Expression): Xuất dữ liệu trực tiếp ra HTML.

Đối tượng ẩn (Implicit Objects): request (nhận dữ liệu), response (phản hồi), session (lưu phiên làm việc), application (dữ liệu dùng chung toàn ứng dụng).

1.2. Cơ chế Giỏ hàng (Shopping Cart)

Sử dụng Session để lưu trữ trạng thái vì HTTP là giao thức không trạng thái (stateless).

Quy trình: Khi "Add to Cart", Server lấy ID sản phẩm -> Kiểm tra giỏ hàng trong Session -> Cập nhật số lượng hoặc thêm mới vào List/Map -> Lưu lại vào Session.



🔵 PHẦN 2: SPRING FRAMEWORK 6.X (CỐT LÕI)

2.1. Inversion of Control (IoC) & Dependency Injection (DI)

IoC: Là nguyên lý chuyển quyền quản lý đối tượng từ lập trình viên sang Framework (Spring Container).

DI: Là phương pháp thực hiện IoC. Spring "tiêm" các phụ thuộc vào class.

Constructor Injection: Tiêm qua hàm tạo (An toàn nhất, hỗ trợ biến final).

Setter Injection: Tiêm qua hàm set (Dùng cho phụ thuộc tùy chọn).

Field Injection: Dùng @Autowired trực tiếp (Nhanh nhưng khó test).

2.2. Spring Bean & Scopes

Bean: Là đối tượng được Spring Container khởi tạo và quản lý.

Scopes (Phạm vi):

Singleton (Mặc định): Chỉ có 1 instance duy nhất cho toàn bộ ứng dụng.

Prototype: Tạo mới instance mỗi khi được yêu cầu.

Request/Session: Tồn tại theo một yêu cầu HTTP hoặc một phiên làm việc.

2.3. Vòng đời của Bean (Bean Lifecycle)

Instantiate: Khởi tạo đối tượng.

Populate Properties: Tiêm phụ thuộc (DI).

Aware Interfaces: Nhận biết các tài nguyên của Spring (BeanName, BeanFactory).

BeanPostProcessor: Xử lý trước và sau khi khởi tạo.

Initialization: Gọi hàm @PostConstruct hoặc afterPropertiesSet().

Ready: Bean sẵn sàng hoạt động.

Destruction: Gọi hàm @PreDestroy trước khi container đóng.



🟡 PHẦN 3: SPRING BOOT 3.X

Mục tiêu: Đơn giản hóa cấu hình (Opinionated), giúp tạo ứng dụng Standalone có thể chạy ngay (với lệnh java -jar).

Auto-configuration: Tự động cấu hình các Bean dựa trên các thư viện (starter) có trong Classpath.

Starter Projects: Các gói phụ thuộc gom sẵn (vd: spring-boot-starter-web) giúp quản lý dependency dễ dàng hơn.

Embedded Server: Tích hợp sẵn Server (thường là Tomcat) vào bên trong file JAR, không cần cài server rời.



🟠 PHẦN 4: THAO TÁC DỮ LIỆU (JPA & SPRING DATA JPA)

ORM (Object-Relational Mapping): Kỹ thuật ánh xạ dữ liệu từ Database sang đối tượng Java.

JPQL (Java Persistence Query Language): Truy vấn dựa trên các lớp Entity. (Vd: SELECT u FROM User u).

Native Query: SQL thuần túy của từng hệ quản trị DB (MySQL, Oracle). Dùng khi cần tối ưu hiệu năng hoặc xử lý logic đặc thù mà JPQL không hỗ trợ.

Fetch Type:

LAZY: Trì hoãn việc tải dữ liệu liên quan cho đến khi được truy cập (Tối ưu bộ nhớ).

EAGER: Tải tất cả dữ liệu liên quan ngay lập tức.

Quan hệ: @OneToMany (1-N), @ManyToOne (N-1), @ManyToMany (N-N).



🔴 PHẦN 5: SPRING WEB & RESTFUL API

Quy trình Request: DispatcherServlet -> HandlerMapping -> Controller -> Service -> Repository -> Trả về View/Data.

Annotation:

@Controller: Trả về giao diện (HTML/JSP).

@RestController: Trả về dữ liệu thô (thường là JSON).

@PathVariable: Lấy giá trị trên đường dẫn URL (/users/{id}).

@RequestParam: Lấy tham số sau dấu hỏi (?name=An).

@RequestBody: Chuyển đổi nội dung JSON từ Request Body sang đối tượng Java.

HATEOAS: Cung cấp các đường link hướng dẫn trong phản hồi để Client biết các thao tác tiếp theo (vd: link tới trang sau, link xóa...).

📑 PHẦN 8: CÁC CÂU HỎI LÝ THUYẾT TRỌNG TÂM (FAQs)

Phân biệt @Primary và @Qualifier?

@Primary đánh dấu Bean ưu tiên mặc định. @Qualifier dùng để chỉ đích danh tên Bean cụ thể khi tiêm.

Tại sao Spring Boot không cần file web.xml?

Vì nó sử dụng cấu hình Java (@Configuration) và cơ chế Auto-configuration để tự động đăng ký các Servlet/Filter.

Lợi ích của Spring Data JPA?

Giảm mã lặp (Boilerplate code). Chỉ cần khai báo Interface, Spring sẽ tự động tạo các phương thức CRUD cơ bản
